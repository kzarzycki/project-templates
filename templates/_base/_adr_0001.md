# 1. Record architecture decisions

- Status: accepted

## Context

We need to record the architectural decisions made on this project, so the
reasoning behind them survives turnover and stays open to revisiting.

## Decision

We use Architecture Decision Records, as first described by Michael Nygard, in a
lightweight MADR format. One file per decision under `docs/adr/`, numbered in
sequence (`0002-...`, `0003-...`).

## Consequences

Decisions are reviewable in pull requests and discoverable in one place. Each
significant decision adds a numbered record here.
